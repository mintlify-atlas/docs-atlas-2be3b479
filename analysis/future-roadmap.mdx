## Overview

The Claude Code v2.1.88 source contains extensive evidence of upcoming models, an autonomous agent mode, voice input, a virtual pet system, and over a dozen unreleased tools. This page documents what the architecture reveals about each direction.

## Next model: Numbat

The most concrete evidence of the next model launch appears as a code comment:

```typescript
// src/constants/prompts.ts:402
// @[MODEL LAUNCH]: Remove this section when we launch numbat.
```

**Numbat** (袋食蚁兽) is the codename for an upcoming model. The comment indicates that the current output efficiency section will be revised or removed at Numbat's launch, suggesting the new model may have better native output control that makes the existing workarounds unnecessary.

### Future version numbers

The undercover mode instructions contain explicit references to unreleased version numbers that must not leak (`src/utils/undercover.ts:49`):

```typescript
- Unreleased model version numbers (e.g., opus-4-7, sonnet-4-8)
```

**Opus 4.7** and **Sonnet 4.8** are both in active development.

### MODEL LAUNCH checklist

The codebase contains 20+ `@[MODEL LAUNCH]` markers indicating everything that must be updated at each model release:

- Default model names (`FRONTIER_MODEL_NAME`)
- Model family IDs
- Knowledge cutoff dates
- Pricing tables
- Context window configurations
- Thinking mode support flags
- Display name mappings
- Migration scripts
- `tengu_ant_model_override` flag update
- `scripts/excluded-strings.txt` update with new codename

## KAIROS — autonomous agent mode

KAIROS is the largest unreleased feature. It transforms Claude Code from a reactive assistant into a proactive autonomous agent that operates independently of user input.

### System prompt

```text
// src/constants/prompts.ts:860-913

You are running autonomously.
You will receive <tick> prompts that keep you alive between turns.
If you have nothing useful to do, call SleepTool.
Bias toward action — read files, make changes, commit without asking.

## Terminal focus
- Unfocused: The user is away. Lean heavily into autonomous action.
- Focused: The user is watching. Be more collaborative.
```

The `<tick>` heartbeat mechanism keeps the agent running between turns, and the terminal focus state adjusts how aggressively the agent acts when no human is watching.

### Associated tools

| Tool | Feature flag | Purpose |
|------|-------------|---------|
| `SleepTool` | `KAIROS` / `PROACTIVE` | Control pacing between autonomous actions |
| `SendUserFileTool` | `KAIROS` | Proactively send files to users |
| `PushNotificationTool` | `KAIROS` / `KAIROS_PUSH_NOTIFICATION` | Push notifications to user devices |
| `SubscribePRTool` | `KAIROS_GITHUB_WEBHOOKS` | Subscribe to GitHub PR webhook events |
| `BriefTool` | `KAIROS_BRIEF` | Proactive status updates |

### Behavior

- Operates on `<tick>` heartbeat prompts to stay alive between user turns
- Adjusts autonomy level based on terminal focus state (user present vs away)
- Can commit, push, and make decisions independently without user confirmation
- Sends proactive push notifications and status updates to user devices
- Monitors GitHub PRs via webhooks for autonomous review and response

## Voice mode

Push-to-talk voice input is fully implemented in the source but gated behind the `VOICE_MODE` feature flag:

```typescript
// src/voice/voiceModeEnabled.ts
// Connects to Anthropic's voice_stream WebSocket endpoint
// Uses conversation_engine backed models for speech-to-text
// Hold-to-talk: hold keybinding to record, release to submit
```

| Property | Value |
|----------|-------|
| Input method | Hold-to-talk (hold keybinding to record, release to submit) |
| Transport | `voice_stream` WebSocket |
| Security | mTLS for WebSocket connections |
| Auth | OAuth-only — no API key, Bedrock, or Vertex support |
| Killswitch | `tengu_amber_quartz_disabled` |

## Coordinator mode

A multi-agent coordination system is implemented under a feature flag:

```typescript
// src/coordinator/coordinatorMode.ts
// Feature flag: COORDINATOR_MODE
```

Coordinator mode enables task execution across multiple Claude Code agents with shared state and inter-agent messaging. It is distinct from the agent nesting available to internal users — coordinator mode is designed for structured multi-agent workflows.

## Dream task

A background memory consolidation subagent runs during idle time:

```text
// src/tasks/DreamTask/
// Auto-dreaming feature that works in the background
// Controlled by 'tengu_onyx_plover' feature flag
```

When enabled via `tengu_onyx_plover`, the Dream Task processes and consolidates memories autonomously during sessions, without user initiation. This complements the KAIROS autonomous mode by providing continuous background context maintenance.

## Three directions

The architecture points clearly toward three major strategic directions:

1. **New models**: Numbat is the next launch; Opus 4.7 and Sonnet 4.8 are in parallel development
2. **Autonomous agent (KAIROS)**: Unattended operation, `<tick>` heartbeats, proactive commits, push notifications, GitHub PR monitoring
3. **Multi-modal**: Voice input fully implemented, browser automation tool ready, workflow scripting and coordinator mode in progress

Claude Code is evolving from a coding assistant into an always-on autonomous development agent.
